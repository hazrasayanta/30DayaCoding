## **5. Props vs State (Key Differences)**

| Feature                            | Props                           | State                                    |
| ---------------------------------- | ------------------------------- | ---------------------------------------- |
| **Mutability**               | Immutable (Read-only)           | Mutable (Can change)                     |
| **Who Manages It?**          | Parent Component                | Component Itself                         |
| **Used For**                 | Passing data between components | Managing dynamic data inside a component |
| **Changes Cause Re-render?** | Yes                             | Yes                                      |

## **6. What is State in React?**

âœ”ï¸ **State is a built-in React object** used to store **dynamic** component data.

âœ”ï¸ **Unlike props, state is mutable** (can be changed).

âœ”ï¸ **When state updates, React re-renders the component.**

### âœ… **State Example in Functional Component (useState Hook)**

```jsx
import { useState } from "react";

function Counter() {
    const [count, setCount] = useState(0); // State variable

    return (
        <div>
            <h1>Count: {count}</h1>
            <button onClick={() => setCount(count + 1)}>Increase</button>
        </div>
    );
}
```

ğŸ”¹ **The `count` state changes when the button is clicked, re-rendering the component.**

## **7. State in Class Components (`this.state`)**

âœ”ï¸ **State is initialized in the constructor.**

âœ”ï¸ **Updated using `this.setState()` method.**

### âœ… **State Example in Class Component**

```jsx

    constructor(props) {
        super(props);
        this.state = { count: 0 };
    }

    increment = () => {
        this.setState({ count: this.state.count + 1 });
    };

    render() {
        return (
            <div>
                <h1>Count: {this.state.count}</h1>
                <button onClick={this.increment}>Increase</button>
            </div>
        );
    }
}
```

ğŸ”¹ **`this.setState()` triggers re-render with updated state.**


## **11. State with Objects & Arrays**

âœ”ï¸ **Always copy the previous state when updating objects or arrays!**

