## **1. Why Optimize Performance in React?**

‚úÖ To improve **rendering speed** and  **user experience** .

‚úÖ To **avoid unnecessary re-renders** that slow down UI updates.

‚úÖ To **reduce memory usage** and improve app efficiency.

‚úÖ To ensure smooth performance on  **low-end devices** .

---

## **2. Common Performance Issues in React**

üö® **Potential Bottlenecks:**

‚ùå Unnecessary  **component re-renders** .

‚ùå **Expensive computations** running on every render.

‚ùå Large lists causing  **slow rendering** .

‚ùå Inline functions causing **new references** on every render.

‚ùå Unoptimized **API calls** and  **heavy state updates** .

---

## **3. Key React Performance Optimization Techniques**

### **1Ô∏è‚É£ Using `React.memo()` (Memoization for Functional Components)**

‚úî Prevents re-rendering when  **props remain unchanged** .

‚úî Works like `PureComponent` for class components.

üîπ **Example:**

```js
import React from "react";

const ExpensiveComponent = ({ data }) => {
    console.log("Rendering Expensive Component");
    return <div>Data: {data}</div>;
};

export default React.memo(ExpensiveComponent);
```

‚úÖ Now, `ExpensiveComponent`  **only re-renders when `data` changes** .

---

### **2Ô∏è‚É£ Using `useMemo()` for Expensive Calculations**

‚úî `useMemo()` **caches** the result of expensive calculations.

‚úî It  **only recalculates when dependencies change** .

üîπ **Example:**

```js
import React, { useState, useMemo } from "react";

function ExpensiveCalculation({ number }) {
    const squaredNumber = useMemo(() => {
        console.log("Calculating...");
        return number * number;
    }, [number]);

    return <div>Squared Number: {squaredNumber}</div>;
}
```

‚úÖ Prevents re-computation **on every render** unless `number` changes.

---

### **3Ô∏è‚É£ Using `useCallback()` to Prevent Unnecessary Function Recreation**

‚úî Prevents **functions from being recreated** on each render.

‚úî Useful when passing functions to  **child components** .

üîπ **Example:**

```js
import React, { useState, useCallback } from "react";

function ParentComponent() {
    const [count, setCount] = useState(0);

    const increment = useCallback(() => {
        setCount((prevCount) => prevCount + 1);
    }, []);

    return <ChildComponent increment={increment} />;
}

function ChildComponent({ increment }) {
    console.log("Child Component Rendered");
    return <button onClick={increment}>Increase</button>;
}
```

‚úÖ `useCallback()` ensures the `increment` function  **doesn't change on every render** .

---

### **4Ô∏è‚É£ Avoiding Inline Functions & Objects in JSX**

üö® **Bad Practice (Causes New Reference Every Render)**

```js
<button onClick={() => console.log("Clicked!")}>Click Me</button>;
```

üöÄ **Optimized Approach**

```js
const handleClick = useCallback(() => console.log("Clicked!"), []);
<button onClick={handleClick}>Click Me</button>;
```

‚úÖ Prevents function re-creation on every render.

---

### **5Ô∏è‚É£ Virtualizing Long Lists with `react-window` or `react-virtualized`**

‚úî Rendering large lists  **slows down performance** .

‚úî **Solution:** Use **windowing** (only render items in viewport).

üîπ **Example using `react-window`:**

```sh
npm install react-window
```

```js
import { FixedSizeList as List } from "react-window";

const Row = ({ index, style }) => <div style={style}>Row {index}</div>;

function App() {
    return <List height={400} width={300} itemSize={50} itemCount={1000}>{Row}</List>;
}
```

‚úÖ Renders  **only visible rows** , improving performance for large lists.

---

### **6Ô∏è‚É£ Using `React.PureComponent` for Class Components**

‚úî **Prevents re-renders** when props/state  **don't change** .

üîπ **Example:**

```js
import React from "react";

class MyComponent extends React.PureComponent {
    render() {
        console.log("Rendered!");
        return <div>{this.props.value}</div>;
    }
}
```

‚úÖ Works like `React.memo()` but for class components.

---

### **7Ô∏è‚É£ Lazy Loading Components with `React.lazy()` & `Suspense`**

‚úî  **Loads components only when needed** , reducing initial bundle size.

üîπ **Example:**

```js
import React, { lazy, Suspense } from "react";

const HeavyComponent = lazy(() => import("./HeavyComponent"));

function App() {
    return (
        <Suspense fallback={<div>Loading...</div>}>
            <HeavyComponent />
        </Suspense>
    );
}
```

‚úÖ Improves initial load speed by  **splitting bundles** .

---

### **8Ô∏è‚É£ Avoiding Unnecessary State Updates**

‚úî Updating state  **triggers re-renders** , so  **minimize state changes** .

‚úî **Example:**

üö® **Bad Practice:**

```js
const [count, setCount] = useState(0);

const handleClick = () => {
    setCount(count + 1);
    setCount(count + 1); // Double update, but React batches updates
};
```

üöÄ **Optimized Approach (Functional Updates)**

```js
const handleClick = () => {
    setCount((prevCount) => prevCount + 1);
};
```

‚úÖ Ensures correct state updates  **without extra re-renders** .

---

### **9Ô∏è‚É£ Optimizing API Calls with `useEffect` Dependencies**

‚úî **Prevent infinite loops** by **setting correct dependencies** in `useEffect()`.

üîπ **Example:**

üö® **Bad Practice (Runs on every render):**

```js
useEffect(() => {
    fetchData();
});
```

üöÄ **Optimized Approach (Runs Only on Mount & When `id` Changes):**

```js
useEffect(() => {
    fetchData();
}, [id]);
```

‚úÖ Prevents  **unnecessary API calls** .

## **4. Summary of Optimization Techniques**

| **Optimization**                    | **Technique**                    |
| ----------------------------------------- | -------------------------------------- |
| **Avoid Unnecessary Re-renders**    | `React.memo()`,`PureComponent`     |
| **Optimize Expensive Computations** | `useMemo()`                          |
| **Prevent Function Recreation**     | `useCallback()`                      |
| **Reduce Component Updates**        | Avoid inline functions/objects         |
| **Optimize Large Lists**            | `react-window`,`react-virtualized` |
| **Lazy Load Components**            | `React.lazy()`,`Suspense`          |
| **Minimize State Updates**          | Functional updates in `setState()`   |
| **Optimize API Calls**              | Use proper `useEffect`dependencies   |
