## **3. Setting Up React Router**

âœ” `BrowserRouter` is used to enable routing in a React app.

âœ” `Routes` contains multiple `Route` components.

### âœ… **Basic Example (App.js)**

```jsx
import React from "react";
import { BrowserRouter as Router, Routes, Route } from "react-router-dom";
import Home from "./Home";
import About from "./About";
import Contact from "./Contact";

function App() {
    return (
        <Router>
            <Routes>
                <Route path="/" element={<Home />} />
                <Route path="/about" element={<About />} />
                <Route path="/contact" element={<Contact />} />
            </Routes>
        </Router>
    );
}

export default App;
```

ðŸ”¹ `path="/"` â†’ Defines the  **URL path** .

ðŸ”¹ `element={<Home />}` â†’ Component to render when this route is accessed.

ðŸ”¹ **React Router does NOT reload the page!**

## **4. Navigation with `Link` and `NavLink`**

âœ” Instead of `<a href="">`, use **`Link`** to prevent full-page reloads.

### âœ… **Using `Link`**

```jsx

function Navbar() {
    return (
        <nav>
            <Link to="/">Home</Link> | 
            <Link to="/about">About</Link> | 
            <Link to="/contact">Contact</Link>
        </nav>
    );
}
```

ðŸ”¹ `to="/"` works like `href` but  **without refreshing the page** .

### âœ… **Using `NavLink` (for Active Styling)**

```jsx

    return (
        <nav>
            <NavLink to="/" activeClassName="active">Home</NavLink> | 
            <NavLink to="/about" activeClassName="active">About</NavLink> | 
            <NavLink to="/contact" activeClassName="active">Contact</NavLink>
        </nav>
    );
}
```

ðŸ”¹ `NavLink` automatically adds an `active` class when the route matches.

## **5. Handling 404 (Not Found) Pages**

âœ” Use `"*"` as the last route to handle unknown URLs.

### âœ… **404 Page Example**

```jsx

    return <h2>404 - Page Not Found</h2>;
}

function App() {
    return (
        <Router>
            <Routes>
                <Route path="/" element={<Home />} />
                <Route path="/about" element={<About />} />
                <Route path="/contact" element={<Contact />} />
                <Route path="*" element={<NotFound />} />
            </Routes>
        </Router>
    );
}
```

ðŸ”¹ **Any unknown URL** will display the `NotFound` component.

## **6. Dynamic Routing (URL Parameters)**

âœ” React Router allows **dynamic parameters** using `:id`.

### âœ… **Example: Displaying User Profiles Dynamically**

ðŸ”¹ URL `/user/123` â†’ Displays `Profile of User 123`.

## **7. Programmatic Navigation (`useNavigate`)**

âœ” **`useNavigate` replaces `useHistory` in React Router v6.**

âœ” Used to navigate **programmatically** after an event (e.g., form submission).

### âœ… **Example: Redirecting After Login**

```jsx

    const navigate = useNavigate();

    const handleLogin = () => {
        // Perform authentication logic here
        navigate("/dashboard"); // Redirect after login
    };

    return <button onClick={handleLogin}>Login</button>;
}
```

ðŸ”¹ `navigate("/dashboard")` redirects the user.

## **9. Protected Routes (Authentication Check)**

âœ” Prevents users from accessing certain pages  **without authentication** .

### âœ… **Example: Creating a Private Route**

```jsx

function PrivateRoute({ children }) {
    const isAuthenticated = false; // Replace with auth logic

    return isAuthenticated ? children : <Navigate to="/login" />;
}
```

ðŸ”¹ Redirects **unauthenticated users** to `/login`.

### âœ… **Using Private Routes**

```jsx

```

ðŸ”¹ Users must be logged in to access `/dashboard`.
