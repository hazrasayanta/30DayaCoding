# **Fetch API & HTTP Requests - Interview Revision Guide** üöÄ

## **1. What is the Fetch API?**

‚úî The **Fetch API** is a modern way to make HTTP requests in JavaScript.

‚úî It **replaces** `XMLHttpRequest (XHR)` and provides a **simpler, more readable** syntax.

‚úî It  **returns a Promise** , allowing us to handle asynchronous requests efficiently.

---

## **2. Basic Fetch Request**

Fetching data from an API (GET request):

```js
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => response.json()) // Convert response to JSON
  .then(data => console.log(data)) // Handle data
  .catch(error => console.error("Error:", error)); // Handle errors
```

‚úÖ `fetch(url)` sends an HTTP request.

‚úÖ `.json()` parses the response body as JSON.

‚úÖ `.catch(error)` handles network errors.

---

## **3. Handling HTTP Methods (GET, POST, PUT, DELETE)**

### **1Ô∏è‚É£ GET Request (Fetching Data)**

```js
fetch("https://jsonplaceholder.typicode.com/posts")
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error("Error:", error));
```

### **2Ô∏è‚É£ POST Request (Sending Data)**

```js
fetch("https://jsonplaceholder.typicode.com/posts", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ title: "New Post", body: "Content here", userId: 1 })
})
  .then(response => response.json())
  .then(data => console.log("Created:", data))
  .catch(error => console.error("Error:", error));
```

‚úÖ **`headers`** define content type.

‚úÖ **`body`** sends data as JSON.

### **3Ô∏è‚É£ PUT Request (Updating Data)**

```js
fetch("https://jsonplaceholder.typicode.com/posts/1", {
  method: "PUT",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ title: "Updated Title", body: "Updated content" })
})
  .then(response => response.json())
  .then(data => console.log("Updated:", data))
  .catch(error => console.error("Error:", error));
```

‚úÖ PUT replaces the entire resource.

### **4Ô∏è‚É£ DELETE Request (Removing Data)**

```js
fetch("https://jsonplaceholder.typicode.com/posts/1", {
  method: "DELETE"
})
  .then(response => console.log("Deleted successfully"))
  .catch(error => console.error("Error:", error));
```

‚úÖ DELETE removes the resource.

---

## **4. Handling Errors in Fetch API**

Fetch **only rejects** on  **network failure** , not on HTTP errors (like `404` or `500`).

We need to manually check `response.ok` for errors.

```js
fetch("https://jsonplaceholder.typicode.com/invalid-url")
  .then(response => {
    if (!response.ok) throw new Error(`HTTP error! Status: ${response.status}`);
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error("Fetch Error:", error.message));
```

‚úÖ `response.ok` is `false` for HTTP errors.

‚úÖ `response.status` provides the status code.

---

## **5. Using Fetch with Async/Await (Recommended)**

### **1Ô∏è‚É£ Basic Async/Await Fetch**

```js
async function fetchData() {
  try {
    let response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
    if (!response.ok) throw new Error(`HTTP Error! Status: ${response.status}`);
    let data = await response.json();
    console.log(data);
  } catch (error) {
    console.error("Error:", error.message);
  }
}
fetchData();
```

‚úÖ **More readable** than `.then()`.

‚úÖ Uses `try...catch` for better error handling.

### **2Ô∏è‚É£ POST Request with Async/Await**

```js
async function createPost() {
  try {
    let response = await fetch("https://jsonplaceholder.typicode.com/posts", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ title: "Async Post", body: "Content here", userId: 1 })
    });

    if (!response.ok) throw new Error(`HTTP Error! Status: ${response.status}`);
    let data = await response.json();
    console.log("Created:", data);
  } catch (error) {
    console.error("Error:", error.message);
  }
}
createPost();
```

‚úÖ `await` pauses execution until the request completes.

‚úÖ **`try...catch`** handles both network and HTTP errors.

## **6. Abort Fetch Requests (Avoid Memory Leaks)**

Use **AbortController** to  **cancel ongoing requests** .

```js
const controller = new AbortController();
const signal = controller.signal;

fetch("https://jsonplaceholder.typicode.com/posts", { signal })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => {
    if (error.name === "AbortError") {
      console.log("Fetch request was aborted!");
    } else {
      console.error("Fetch Error:", error.message);
    }
  });

// Abort the request before it completes
setTimeout(() => controller.abort(), 100);
```

‚úÖ Prevents unwanted requests from using resources.

‚úÖ **Useful in React when unmounting components.**

---

## **7. Handling Multiple Fetch Requests (Promise.all)**

When making multiple API calls, **`Promise.all()`** ensures all requests complete.

```js
Promise.all([
  fetch("https://jsonplaceholder.typicode.com/posts/1").then(res => res.json()),
  fetch("https://jsonplaceholder.typicode.com/users/1").then(res => res.json())
])
  .then(([post, user]) => {
    console.log("Post:", post);
    console.log("User:", user);
  })
  .catch(error => console.error("Error:", error));
```

‚úÖ Runs multiple requests  **in parallel** .

‚úÖ **Faster** than running requests sequentially.

### ‚úÖ **Q1: What is the Fetch API? How is it different from XHR?**

## **1Ô∏è‚É£ What is the Fetch API?**

The **Fetch API** is a modern JavaScript interface used to make HTTP requests (GET, POST, etc.).

It is  **promise-based** , making it easier to use than the older `XMLHttpRequest (XHR)`.

### **Example: Fetch API (Basic Usage)**

```javascript
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => response.json())  // Convert response to JSON
  .then(data => console.log(data))   // Handle the data
  .catch(error => console.error("Error:", error)); // Handle errors
```

‚úÖ **Key Features of Fetch API:**

* **Promise-based** (no need for callbacks).
* **Supports async/await** for cleaner code.
* **Works with modern JavaScript features like Streams and Service Workers** .
* **More readable syntax** than XHR.

---

## **2Ô∏è‚É£ How is Fetch API Different from XHR?**

| Feature                              | Fetch API üÜï                     | XHR (XMLHttpRequest) üÜñ          |
| ------------------------------------ | -------------------------------- | -------------------------------- |
| **Syntax**                     | Clean & readable (Promise-based) | Verbose & callback-based         |
| **Chaining**                   | `.then()`,`.catch()`         | Nested callbacks (callback hell) |
| **Async/Await Support**        | ‚úÖ Yes                           | ‚ùå No                            |
| **Streaming Response**         | ‚úÖ Yes (ReadableStream)          | ‚ùå No                            |
| **Error Handling**             | Only rejects on network failure  | Must manually check `status`   |
| **Supports JSON Parsing**      | ‚úÖ Yes (`.json()`)             | ‚ùå No (manual `JSON.parse()`)  |
| **Works with Service Workers** | ‚úÖ Yes                           | ‚ùå No                            |
| **CORS Handling**              | ‚úÖ Yes (Built-in)                | ‚ùå More complex                  |

üëâ The **Response object** contains details like status, headers, and body.

### ‚úÖ **Q4: How do you send a POST request using Fetch?**

## **1Ô∏è‚É£ What is a POST Request?**

A **POST request** is used to **send data** to a server (e.g., creating a new user, submitting a form).

**Key Differences from GET:**

| Feature             | GET                          | POST            |
| ------------------- | ---------------------------- | --------------- |
| **Data Sent** | In URL (query params)        | In request body |

| **Use Cases** | Fetching data                | Submitting data |

---

## **2Ô∏è‚É£ Basic `fetch()` Syntax for a POST Request**

```javascript

  method: "POST",  // üëà Specifies POST request
  headers: {
    "Content-Type": "application/json",  // üëà Tells the server we're sending JSON
  },
  body: JSON.stringify({
    title: "JavaScript Fetch",
    body: "This is a test post",
    userId: 1,
  }),
})
  .then(response => response.json()) // üëà Convert response to JSON
  .then(data => console.log("Success:", data)) // üëà Handle response data
  .catch(error => console.error("Error:", error)); // üëà Handle errors
```

‚úÖ **Fix:** Always set `Content-Type: application/json` when sending JSON.

‚úÖ **Fix:** Convert the `body` to a JSON string using `JSON.stringify()`.

---

## **3Ô∏è‚É£ Using `async/await` for Better Readability**

```javascript

    console.error("Error:", error.message);
  }

sendPostRequest();
```

‚úÖ **More readable than `.then()` chaining!**

‚úÖ **Proper error handling with `try...catch`!**

### ‚úÖ **Q5: What is `async/await` and how does it improve Fetch requests?**

---

## **1Ô∏è‚É£ What is `async/await`?**

* `async/await` is a modern **ES2017+ feature** in JavaScript for handling asynchronous operations.
* It  **simplifies promise-based code** , making it look more like synchronous code.
* It helps avoid **callback hell** and  **`.then()` chaining** .

---

## **2Ô∏è‚É£ How `async/await` Works**

* `async` before a function makes it return a  **Promise** .
* `await` pauses execution  **until the Promise resolves** .

### üîπ **Example Without `async/await` (Using `.then()`)**

```javascript
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => response.json())
  .then(data => console.log("Post Title:", data.title))
  .catch(error => console.error("Error:", error));
```

üî¥ **Problem:** Multiple `.then()` calls make it harder to read and debug.

---

## **3Ô∏è‚É£ Fetch Requests Using `async/await`**

### ‚úÖ **With `async/await`**

```javascript
async function fetchPost() {
  try {
    let response = await fetch("https://jsonplaceholder.typicode.com/posts/1");

    if (!response.ok) {
      throw new Error(`HTTP error! Status: ${response.status}`);
    }

    let data = await response.json();
    console.log("Post Title:", data.title);
  } catch (error) {
    console.error("Error:", error.message);
  }
}

fetchPost();
```

‚úÖ **Fix:** No more `.then()` chaining.

‚úÖ **Fix:** Readable like synchronous code.

‚úÖ **Fix:** `try...catch` handles errors properly.

---

## **4Ô∏è‚É£ Why `async/await` is Better for Fetch Requests**

| Feature                  | `.then()`                                   | `async/await`                          |
| ------------------------ | --------------------------------------------- | ---------------------------------------- |
| **Readability**    | Harder to read with multiple `.then()`calls | Clean & structured like synchronous code |
| **Error Handling** | Need `.catch()`separately                   | Uses `try...catch`for better control   |
| **Debugging**      | Difficult due to `.then()`nesting           | Easier, as errors are caught normally    |
| **Chaining**       | Becomes complex for multiple requests         | Simpler with multiple `await`calls     |

---

## **5Ô∏è‚É£ Example: Fetching Multiple API Requests**

```javascript
async function fetchMultiplePosts() {
  try {
    let response1 = await fetch("https://jsonplaceholder.typicode.com/posts/1");
    let data1 = await response1.json();

    let response2 = await fetch("https://jsonplaceholder.typicode.com/posts/2");
    let data2 = await response2.json();

    console.log("First Post:", data1.title);
    console.log("Second Post:", data2.title);
  } catch (error) {
    console.error("Error fetching posts:", error.message);
  }
}

fetchMultiplePosts();
```

‚úÖ **Fix:** No nested `.then()` calls.

‚úÖ **Fix:** Executes requests in sequence.

---

## **6Ô∏è‚É£ Fetch with `async/await` and `Promise.all` (Parallel Requests)**

```javascript
async function fetchAllPosts() {
  try {
    let urls = [
      "https://jsonplaceholder.typicode.com/posts/1",
      "https://jsonplaceholder.typicode.com/posts/2",
    ];

    let responses = await Promise.all(urls.map(url => fetch(url)));
    let data = await Promise.all(responses.map(res => res.json()));

    console.log("Posts:", data);
  } catch (error) {
    console.error("Error fetching posts:", error.message);
  }
}

fetchAllPosts();
```

‚úÖ **Fix:** Uses `Promise.all()` to fetch multiple APIs  **in parallel** .

‚úÖ **Fix:** Faster than sequential `await` calls.

---

## **üéØ Final Summary (For Interviews)**

‚úî `async/await` makes asynchronous code easier to read and write.

‚úî Works with `fetch()` to remove `.then()` chaining.

‚úî Handles errors using `try...catch`.

‚úî Can be used with `Promise.all()` for parallel API calls.

üöÄ **Conclusion:** `async/await` makes Fetch requests more efficient and readable!



### ‚úÖ **Q6: How can you cancel an ongoing fetch request?**

---

## **1Ô∏è‚É£ Why Cancel a Fetch Request?**

Sometimes, you may need to cancel a fetch request, for example:

‚úî The user navigates away before the request completes.

‚úî A new request replaces an older one (e.g., live search suggestions).

‚úî To avoid unnecessary network usage.

---

## **2Ô∏è‚É£ Using `AbortController` to Cancel Fetch Requests**

JavaScript provides the  **`AbortController` API** , which allows us to cancel a fetch request before it completes.

### üîπ **Example: Cancelling a Fetch Request**

```javascript
const controller = new AbortController();
const signal = controller.signal;

async function fetchData() {
  try {
    let response = await fetch("https://jsonplaceholder.typicode.com/posts/1", { signal });

    let data = await response.json();
    console.log("Fetched Data:", data);
  } catch (error) {
    if (error.name === "AbortError") {
      console.log("Fetch request was canceled!");
    } else {
      console.error("Fetch error:", error);
    }
  }
}

// Start the request
fetchData();

// Cancel the request after 2 seconds
setTimeout(() => controller.abort(), 2000);
```

‚úÖ **Fix:** Uses `AbortController` to cancel the request.

‚úÖ **Fix:** Handles the `"AbortError"` in `catch` to prevent unnecessary logs.

---

## **3Ô∏è‚É£ Cancelling Fetch in Real-World Scenarios**

### **üîπ Use Case: Canceling Previous Requests in Search Input**

When users type in a search box, we want to cancel previous fetch requests and only process the latest one.

```javascript
let controller;

async function search(query) {
  if (controller) {
    controller.abort(); // Cancel the previous request
  }

  controller = new AbortController();
  const signal = controller.signal;

  try {
    let response = await fetch(`https://api.example.com/search?q=${query}`, { signal });
    let data = await response.json();
    console.log("Search results:", data);
  } catch (error) {
    if (error.name === "AbortError") {
      console.log("Previous search request canceled!");
    } else {
      console.error("Fetch error:", error);
    }
  }
}

// Simulating user typing
search("JavaScript");
setTimeout(() => search("JavaScript Fetch API"), 500); // Cancels previous request
```

‚úÖ **Fix:** Cancels previous search requests when the user types.

‚úÖ **Fix:** Prevents unnecessary network requests.

---

## **4Ô∏è‚É£ Cancelling Multiple Fetch Requests**

If multiple fetch requests need to be canceled together,  **use a shared `AbortController`** .

```javascript
const controller = new AbortController();
const signal = controller.signal;

async function fetchUsers() {
  return fetch("https://jsonplaceholder.typicode.com/users", { signal });
}

async function fetchPosts() {
  return fetch("https://jsonplaceholder.typicode.com/posts", { signal });
}

// Start multiple fetch requests
fetchUsers();
fetchPosts();

// Cancel all requests after 3 seconds
setTimeout(() => controller.abort(), 3000);
```

‚úÖ **Fix:** Cancels multiple ongoing requests together.

---

## **üéØ Final Summary (For Interviews)**

‚úî Use **`AbortController`** to cancel fetch requests.

‚úî Attach **`signal`** to `fetch()` for controllable cancellation.

‚úî Handle the `"AbortError"` in `catch` to prevent unwanted logs.

‚úî Useful in  **search inputs** ,  **navigation changes** , and  **optimizing performance** .

üöÄ **Conclusion:** Cancelling fetch requests is essential for efficient network handling!



### ‚úÖ **Q7: How do you fetch multiple APIs in parallel?**

---

## **1Ô∏è‚É£ Why Fetch Multiple APIs in Parallel?**

Fetching multiple APIs in parallel improves performance by executing all requests simultaneously instead of waiting for one to finish before starting another.

üîπ **Use Cases:**

‚úî Fetching **user details** and **user posts** at the same time.

‚úî Fetching **multiple endpoints** in a dashboard.

‚úî Fetching **data from different microservices** simultaneously.

---

## **2Ô∏è‚É£ Using `Promise.all()` for Parallel Requests**

The **best way** to fetch multiple APIs in parallel is to use `Promise.all()`. It executes all promises together and waits for all of them to complete.

### **üîπ Example: Fetching User Data and Posts Together**

```javascript
async function fetchUserData() {
  const userPromise = fetch("https://jsonplaceholder.typicode.com/users/1").then(res => res.json());
  const postsPromise = fetch("https://jsonplaceholder.typicode.com/posts?userId=1").then(res => res.json());

  try {
    const [user, posts] = await Promise.all([userPromise, postsPromise]);
    console.log("User Data:", user);
    console.log("User Posts:", posts);
  } catch (error) {
    console.error("Error fetching data:", error);
  }
}

fetchUserData();
```

‚úÖ **Fix:** Uses `Promise.all()` to run both fetches in parallel.

‚úÖ **Fix:** Improves performance by fetching both at the same time.

‚úÖ **Fix:** Handles errors properly using `try...catch`.

---

## **3Ô∏è‚É£ Handling Errors in `Promise.all()`**

If  **one request fails** , `Promise.all()` **rejects all** promises. To handle errors separately, use `.catch()` on each promise.

### **üîπ Example: Handling Errors Individually**

```javascript
async function fetchMultipleAPIs() {
  const userPromise = fetch("https://jsonplaceholder.typicode.com/users/1").then(res => res.json()).catch(err => console.error("User fetch failed:", err));
  const postsPromise = fetch("https://jsonplaceholder.typicode.com/posts?userId=1").then(res => res.json()).catch(err => console.error("Posts fetch failed:", err));

  const [user, posts] = await Promise.all([userPromise, postsPromise]);
  console.log("User:", user);
  console.log("Posts:", posts);
}

fetchMultipleAPIs();
```

‚úÖ **Fix:** Uses `.catch()` inside each promise to prevent total failure.

‚úÖ **Fix:** Ensures one failure doesn‚Äôt stop other requests.

---

## **4Ô∏è‚É£ Using `Promise.allSettled()` for Better Error Handling**

If we want  **all requests to complete** , even if some fail, use `Promise.allSettled()`.

### **üîπ Example: Using `Promise.allSettled()`**

```javascript
async function fetchWithAllSettled() {
  const responses = await Promise.allSettled([
    fetch("https://jsonplaceholder.typicode.com/users/1").then(res => res.json()),
    fetch("https://jsonplaceholder.typicode.com/posts?userId=1").then(res => res.json()),
    fetch("https://jsonplaceholder.typicode.com/invalid-url").then(res => res.json()) // Invalid URL
  ]);

  responses.forEach((result, index) => {
    if (result.status === "fulfilled") {
      console.log(`API ${index + 1} Success:`, result.value);
    } else {
      console.error(`API ${index + 1} Failed:`, result.reason);
    }
  });
}

fetchWithAllSettled();
```

‚úÖ **Fix:** Ensures all requests run, even if some fail.

‚úÖ **Fix:** Uses `status === "fulfilled"` to check success.

---

## **5Ô∏è‚É£ Using `Promise.race()` for the Fastest Response**

If you want to  **fetch multiple APIs but only use the fastest response** , use `Promise.race()`.

### **üîπ Example: Using `Promise.race()`**

```javascript
async function fetchFastestAPI() {
  try {
    const response = await Promise.race([
      fetch("https://jsonplaceholder.typicode.com/users/1").then(res => res.json()),
      fetch("https://jsonplaceholder.typicode.com/posts/1").then(res => res.json())
    ]);
    console.log("Fastest Response:", response);
  } catch (error) {
    console.error("Error:", error);
  }
}

fetchFastestAPI();
```

‚úÖ **Fix:** Returns only the  **fastest API response** .

‚úÖ **Fix:** Useful for **redundant API calls** where any response is acceptable.

---

## **üéØ Final Summary (For Interviews)**

‚úî **`Promise.all()`** ‚Üí Best for parallel API requests (Fast & efficient).

‚úî **`Promise.allSettled()`** ‚Üí Ensures all requests complete, even if some fail.

‚úî **`Promise.race()`** ‚Üí Returns only the fastest response.

‚úî **Handle errors properly** ‚Üí Use `try...catch` or `.catch()` inside individual promises.

üöÄ **Conclusion:** Parallel fetching improves performance and is essential for  **real-world applications** !



### ‚úÖ **Q8: What is the difference between Fetch API and Axios?**

Both **Fetch API** and **Axios** are used to make HTTP requests in JavaScript, but they have key differences in syntax, features, and error handling.

---

## **1Ô∏è‚É£ Fetch API vs. Axios: Quick Comparison**

| Feature                        | Fetch API (Built-in)        | Axios (3rd-Party Library)      |
| ------------------------------ | --------------------------- | ------------------------------ |
| **Ease of Use**          | More verbose                | Simpler syntax                 |
| **Error Handling**       | Requires manual checking    | Handles errors automatically   |
| **Response Parsing**     | Needs `.json()`conversion | Auto-converts JSON             |
| **Request Cancellation** | Uses `AbortController`    | Uses built-in `CancelToken`  |
| **Timeout Handling**     | Requires manual setup       | Built-in timeout support       |
| **Browser Support**      | Modern browsers only        | Works in older browsers too    |
| **Interceptors**         | Not available               | Available for request/response |

---

## **2Ô∏è‚É£ Fetch API: How It Works**

Fetch is a built-in JavaScript function for making network requests.

### **üîπ Example: Fetching Data with Fetch API**

```javascript
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP Error: ${response.status}`);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error("Fetch Error:", error));
```

‚úÖ  **Requires explicit error handling (`if (!response.ok)`)** .

‚úÖ **Requires `.json()` conversion** to extract data.

---

## **3Ô∏è‚É£ Axios: How It Works**

Axios is a third-party HTTP client that simplifies requests and responses.

### **üîπ Example: Fetching Data with Axios**

```javascript
axios.get("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => console.log(response.data))
  .catch(error => console.error("Axios Error:", error));
```

‚úÖ **Automatically checks for errors** (no need for `if (!response.ok)`).

‚úÖ **Automatically parses JSON** (no need for `.json()`).

---

## **4Ô∏è‚É£ Handling Errors: Fetch vs. Axios**

### **üîπ Fetch API**

```javascript
fetch("https://jsonplaceholder.typicode.com/posts/invalid-url")
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP Error: ${response.status}`);
    }
    return response.json();
  })
  .catch(error => console.error("Fetch Error:", error));
```

‚ùå Fetch **does not reject on HTTP errors** (e.g., 404 or 500).

‚úî You must manually check `response.ok`.

### **üîπ Axios**

```javascript
axios.get("https://jsonplaceholder.typicode.com/posts/invalid-url")
  .then(response => console.log(response.data))
  .catch(error => console.error("Axios Error:", error));
```

‚úî Axios **automatically rejects** HTTP errors (no need for manual checks).

---

## **5Ô∏è‚É£ Request Cancellation: Fetch vs. Axios**

### **üîπ Using `AbortController` with Fetch**

```javascript
const controller = new AbortController();
const signal = controller.signal;

fetch("https://jsonplaceholder.typicode.com/posts", { signal })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error("Fetch Error:", error));

// Cancel the request
controller.abort();
```

‚úÖ Fetch supports cancellation using `AbortController`, but it's  **not built-in** .

### **üîπ Using Axios `CancelToken`**

```javascript
const CancelToken = axios.CancelToken;
const source = CancelToken.source();

axios.get("https://jsonplaceholder.typicode.com/posts", { cancelToken: source.token })
  .then(response => console.log(response.data))
  .catch(error => console.error("Axios Error:", error));

// Cancel the request
source.cancel("Request canceled.");
```

‚úÖ Axios  **has built-in request cancellation** .

---

## **6Ô∏è‚É£ Timeout Handling: Fetch vs. Axios**

### **üîπ Fetch (Manual Timeout Handling)**

Fetch does **not** support timeouts natively. You must use `setTimeout()` manually.

```javascript
const controller = new AbortController();
setTimeout(() => controller.abort(), 5000); // Timeout after 5 seconds

fetch("https://jsonplaceholder.typicode.com/posts", { signal: controller.signal })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error("Fetch Timeout Error:", error));
```

‚ùå  **Fetch has no built-in timeout support** .

‚úî **You must manually abort requests** using `AbortController`.

### **üîπ Axios (Built-in Timeout)**

Axios  **supports timeouts natively** .

```javascript
axios.get("https://jsonplaceholder.typicode.com/posts", { timeout: 5000 })
  .then(response => console.log(response.data))
  .catch(error => console.error("Axios Timeout Error:", error));
```

‚úÖ  **Axios has built-in timeout handling** .

---

## **7Ô∏è‚É£ Request & Response Interceptors (Only in Axios)**

Axios allows you to **modify requests/responses globally** before they are sent or received.

### **üîπ Axios Interceptors Example**

```javascript
axios.interceptors.request.use(config => {
  config.headers.Authorization = "Bearer my-token";
  console.log("Request Sent:", config);
  return config;
});

axios.interceptors.response.use(response => {
  console.log("Response Received:", response);
  return response;
});
```

‚úî  **Useful for adding authentication headers** .

‚úî  **Useful for logging API responses** .

‚ùå  **Fetch API does not support interceptors** .

---

## **üéØ Final Summary (For Interviews)**

| Feature                        | Fetch API                       | Axios                           |
| ------------------------------ | ------------------------------- | ------------------------------- |
| **Built-in?**            | ‚úÖ Yes (Native)                 | ‚ùå No (3rd Party)               |
| **Ease of Use**          | ‚ùå Verbose                      | ‚úÖ Simple                       |
| **Error Handling**       | ‚ùå Needs manual check           | ‚úÖ Automatic                    |
| **JSON Parsing**         | ‚ùå Needs `.json()`            | ‚úÖ Automatic                    |
| **Request Cancellation** | ‚ùå Manual (`AbortController`) | ‚úÖ Built-in (`CancelToken`)   |
| **Timeout Handling**     | ‚ùå Manual (`setTimeout`)      | ‚úÖ Built-in (`timeout`option) |
| **Interceptors**         | ‚ùå No                           | ‚úÖ Yes                          |
| **Browser Support**      | ‚úÖ Modern Browsers              | ‚úÖ Older Browsers Supported     |

üöÄ **Conclusion:**

‚úî Use **Fetch API** when you need a lightweight, built-in solution.

‚úî Use **Axios** when you need  **better error handling, automatic JSON parsing, request cancellation, and timeouts** .
