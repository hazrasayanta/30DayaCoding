### âœ… **Q2: What is the Event Loop and How Does It Work?**

---

### **1ï¸âƒ£ Definition of the Event Loop**

ğŸŒ€ The **Event Loop** is a mechanism in JavaScript that allows asynchronous code to be executed **without blocking** the main thread.

âš¡ It ensures that JavaScript (which is single-threaded) can handle multiple tasks efficiently, such as:

âœ” Handling user input ğŸ–±ï¸

âœ” Fetching data from an API ğŸŒ

### **2ï¸âƒ£ Key Components of the Event Loop**

1. **Call Stack (LIFO - Last In, First Out) ğŸ—ï¸**
   * Holds synchronous code execution.
   * When a function is called, it is **pushed** onto the stack.
   * When execution is complete, it is **popped** off.
2. **Web APIs (for async tasks) ğŸŒ**
   * Handles asynchronous tasks like `setTimeout()`, `fetch()`, and `DOM events`.
   * These tasks run outside the call stack and wait for execution.
3. **Callback Queue (FIFO - First In, First Out) ğŸ“œ**
   * Stores callback functions from async tasks.
   * Waits for the call stack to be empty before execution.
4. **Microtask Queue (Higher Priority) ğŸš€**
   * Stores  **Promises (`.then()`, `catch()`) and `MutationObserver` callbacks** .
   * Executes **before** the Callback Queue if the Call Stack is empty.

### **3ï¸âƒ£ How the Event Loop Works (Step-by-Step) ğŸš€**

1ï¸âƒ£ The **Call Stack** executes synchronous code first.

2ï¸âƒ£ If an **asynchronous task** (e.g., `setTimeout()`, `fetch()`, Promise) is encountered:

* It moves to  **Web APIs** , which handle it.

  3ï¸âƒ£ Once the async task is completed, its **callback** goes to either:
* **Microtask Queue** (if it's a Promise or MutationObserver)
* **Callback Queue** (for `setTimeout()`, `setInterval()`, I/O tasks).

  4ï¸âƒ£ The **Event Loop** checks:
* If the Call Stack is empty, it **picks a task** from the Microtask Queue (higher priority) or Callback Queue and pushes it to the Call Stack.

  5ï¸âƒ£ This repeats infinitely ğŸ”„.

---

### **4ï¸âƒ£ Example of the Event Loop in Action**

```js
  console.log("Step 2 (from setTimeout)");
```


ğŸ“Œ **Explanation:**

1ï¸âƒ£ `"Step 1"` and `"Step 4"` execute first (synchronous).

2ï¸âƒ£ The `setTimeout()` callback moves to the  **Callback Queue** .

3ï¸âƒ£ The Promise callback (`.then()`) goes to the  **Microtask Queue** .

4ï¸âƒ£ The Event Loop prioritizes **Microtask Queue** first (`Step 3`).

5ï¸âƒ£ Finally, the `setTimeout()` callback (`Step 2`) executes.

---

### **5ï¸âƒ£ Key Differences: Callback Queue vs. Microtask Queue**

| Feature                      | Callback Queue (`setTimeout()`)  | Microtask Queue (`Promise.then()`)        |
| ---------------------------- | ---------------------------------- | ------------------------------------------- |
| **Execution Priority** | Lower                              | Higher ğŸš€                                   |
| **Example**            | `setTimeout()`,`setInterval()` | Promises (`.then()`),`queueMicrotask()` |
| **When it runs**       | After the Call Stack is empty      | Before Callback Queue                       |
